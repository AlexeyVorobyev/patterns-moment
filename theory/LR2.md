b. Принцип наследования, переопределение методов и способы вызова переопределенного метода

Наследование — это механизм объектно-ориентированного программирования, который позволяет создавать новый класс на основе существующего. Новый класс называется производным (или подклассом), а существующий — базовым (или суперклассом). Наследование позволяет повторно использовать код и расширять функциональность базового класса.

Переопределение методов — это процесс, при котором производный класс предоставляет свою реализацию метода, который уже определен в базовом классе. Это позволяет изменять или расширять поведение методов.

В Kotlin для переопределения метода в классе необходимо использовать ключевое слово open в базовом классе и override в производном классе. Пример:

open class Animal {
open fun makeSound() {
println("Animal sound")
}
}

class Dog : Animal() {
override fun makeSound() {
println("Bark")
}
}


▎Вызов переопределенного метода

Чтобы вызвать переопределенный метод, можно создать объект производного класса и вызвать метод через этот объект:

val dog = Dog()
dog.makeSound() // Вывод: Bark


Также можно вызвать метод базового класса из переопределенного метода с помощью super:

class Dog : Animal() {
override fun makeSound() {
super.makeSound() // Вызов метода базового класса
println("Bark")
}
}


▎c. Принципы работы конструкторов для наследуемых классов

При наследовании конструкторы работают следующим образом:

1. Первичный конструктор базового класса вызывается автоматически при создании объекта производного класса. Если базовый класс имеет первичный конструктор, то его параметры должны быть переданы при создании объекта производного класса.

2. Вторичный конструктор базового класса также может быть вызван из вторичного конструктора производного класса с помощью super.

Пример:

open class Person(val name: String) {
open fun greet() {
println("Hello, my name is $name")
}
}

class Student(name: String, val studentId: Int) : Person(name) {
override fun greet() {
println("Hello, my name is $name and my student ID is $studentId")
}
}


При создании объекта Student, конструктор Person будет вызван с параметром name:

val student = Student("Alice", 12345)
student.greet() // Вывод: Hello, my name is Alice and my student ID is 12345
